# 预制体创建指南

> 本文档对应脚本架构：参照 `货柜归位大闯关` 分层结构  
> 操作环境：Cocos Creator 3.8.8  
> 项目路径：`/Users/bot/Desktop/XiaoChu/`

---

## 一、准备工作

### 1. 打开项目
1. 启动 **Cocos Creator 3.8.8**
2. 点击「打开其他项目」
3. 选择路径 `/Users/bot/Desktop/XiaoChu/` 并打开

### 2. 确认脚本已加载
打开后，检查 **资源管理器** → `assets/script/` 目录下是否存在以下文件：
```
assets/script/
├── App.ts
├── main.ts
├── const/
│   ├── EventName.ts
│   ├── GameConfig.ts
│   └── ViewName.ts
├── managers/
├── components/
├── games/
└── utils/
```
如有缺失，请先执行 `项目 → 刷新资源`。

---

## 二、第一步：给 Canvas 挂载入口脚本

### 操作步骤
1. 在 **层级管理器** 中点击 `Canvas` 节点
2. 在 **属性检查器** 底部点击「添加组件」
3. 搜索并选择 `MainEntry`（对应 `assets/script/main.ts`）
4. 按 `Ctrl+S`（或 `Cmd+S`）保存场景

> ✅ 完成后，运行游戏时会自动执行 `main.ts` 中的 `onLoad`，打开主菜单。

---

## 三、创建预制体（共 4 个）

> ⚠️ **重要**：所有预制体必须放在 `assets/resources/prefab/view/` 目录下，否则 `ViewManager` 无法加载。

### 3.1 创建目录
在 **资源管理器** 中右键 `assets/resources/` → 新建 → 文件夹，依次创建：
```
assets/resources/prefab/view/
```

---

## 四、预制体 1：homeView（主菜单）

### 4.1 创建节点
1. 在 **层级管理器** 中右键 `Canvas` → 新建 → 2D 对象 → `Sprite`
2. 将该节点重命名为 `homeView`
3. 在 **属性检查器** 中设置：
   - `Sprite → Size Mode`：`CUSTOM`
   - `Sprite → Custom Size`：`750 x 1334`（竖屏分辨率）

### 4.2 添加子节点
在 `homeView` 下新建以下节点（右键 → 新建 → 对应类型）：

| 子节点名称 | 节点类型 | 说明 |
|---|---|---|
| `Img_Title` | Sprite | 游戏标题图片 |
| `Btn_Start` | Sprite + Button | 开始游戏按钮 |
| `Btn_Settings` | Sprite + Button | 设置按钮（可选）|
| `Label_Score` | Label | 显示最高分（可选）|

### 4.3 挂载脚本
1. 选中 `homeView` 节点
2. 在 **属性检查器** 底部点击「添加组件」
3. 搜索 `HomeView`（对应 `assets/script/games/views/HomeView.ts`）
4. 挂载到节点上

### 4.4 按钮事件绑定
1. 选中 `Btn_Start` 节点
2. 在 **属性检查器** → `Button → Click Events` 中：
   - 点击「+」添加一条
   - 将 `homeView` 节点拖入 `Target`
   - 选择 `HomeView` → `onClick_BtnStart`（这是 BaseView 自动绑定的规范）
3. 对 `Btn_Settings` 重复上述步骤，绑定 `onClick_BtnSettings`

### 4.5 保存为预制体
1. 将 `homeView` 节点从 **层级管理器** 拖入 **资源管理器** 的 `assets/resources/prefab/view/` 目录
2. 此时会生成一个 `homeView.prefab` 文件
3. 可以删除场景中的 `homeView` 节点（预制体已保存，运行时由 ViewManager 动态加载）

---

## 五、预制体 2：gameView（游戏主界面）

### 5.1 创建节点
1. 在 **层级管理器** 中右键 `Canvas` → 新建 → 2D 对象 → `Sprite`
2. 重命名为 `gameView`
3. 设置 `Sprite → Custom Size`：`750 x 1334`

### 5.2 添加子节点

| 子节点名称 | 节点类型 | 说明 |
|---|---|---|
| `GameNode` | 空节点（Node） | 游戏根节点，GameManager 在此创建方块 |
| `TouchZone` | Sprite | 触摸区域，覆盖整个屏幕 |
| `Label_Score` | Label | 当前分数显示 |
| `Btn_Pause` | Sprite + Button | 暂停按钮 |

#### TouchZone 设置
1. 选中 `TouchZone` 节点
2. 设置 `Sprite → Custom Size`：`750 x 1334`
3. 设置 `Sprite → Opacity`：`0`（透明，只用于接收触摸）
4. 勾选 `Button` 组件（如不需要 Button 交互，可不添加，由 GameView 直接监听触摸）

### 5.3 挂载脚本
1. 选中 `gameView` 节点
2. 添加组件 `GameView`（对应 `assets/script/games/views/gameView.ts`）

### 5.4 暂停按钮事件
1. 选中 `Btn_Pause`
2. `Button → Click Events` → 绑定 `gameView` 节点 → `GameView` → `onClick_BtnPause`

### 5.5 保存为预制体
将 `gameView` 拖入 `assets/resources/prefab/view/` 生成 `gameView.prefab`，然后删除场景中的实例。

---

## 六、预制体 3：resultView（结算界面）

### 6.1 创建节点
1. 新建 Sprite 节点，重命名为 `resultView`
2. 设置大小 `750 x 1334`

### 6.2 添加子节点

| 子节点名称 | 节点类型 | 说明 |
|---|---|---|
| `Label_FinalScore` | Label | 最终分数 |
| `Label_HighScore` | Label | 最高分（可选）|
| `Btn_Restart` | Sprite + Button | 重新开始 |
| `Btn_Home` | Sprite + Button | 返回主页 |

### 6.3 挂载脚本
添加组件 `ResultView`（对应 `assets/script/games/views/ResultView.ts`）

### 6.4 按钮事件
- `Btn_Restart` → `ResultView` → `onClick_BtnRestart`
- `Btn_Home` → `ResultView` → `onClick_BtnHome`

### 6.5 保存为预制体
拖入 `assets/resources/prefab/view/` → `resultView.prefab`

---

## 七、预制体 4：pauseView（暂停界面）

### 7.1 创建节点
1. 新建 Sprite 节点，重命名为 `pauseView`
2. 设置大小 `750 x 1334`（建议半透明遮罩）

### 7.2 添加子节点

| 子节点名称 | 节点类型 | 说明 |
|---|---|---|
| `Btn_Resume` | Sprite + Button | 继续游戏 |
| `Btn_Restart` | Sprite + Button | 重新开始 |
| `Btn_Home` | Sprite + Button | 返回主页 |

### 7.3 挂载脚本
添加组件 `PauseView`（对应 `assets/script/games/views/PauseView.ts`）

### 7.4 按钮事件
- `Btn_Resume` → `PauseView` → `onClick_BtnResume`
- `Btn_Restart` → `PauseView` → `onClick_BtnRestart`
- `Btn_Home` → `PauseView` → `onClick_BtnHome`

### 7.5 保存为预制体
拖入 `assets/resources/prefab/view/` → `pauseView.prefab`

## 八、验证 ViewName.ts 路径

打开 `assets/script/const/ViewName.ts`，确认路径与预制体位置一致：

```typescript
export const ViewName = {
    homeView: 'prefab/view/homeView',
    gameView: 'prefab/view/gameView',
    resultView: 'prefab/view/resultView',
    pauseView: 'prefab/view/pauseView',
};
```

如果路径不一致，请修改后保存。

## 九、测试运行

### 9.1 保存场景
按 `Ctrl+S`（或 `Cmd+S`）保存当前场景。

### 9.2 点击运行
点击 Cocos Creator 上方的 **播放按钮**（▶），或直接按 `Ctrl+P`（或 `Cmd+P`）。

### 9.3 预期结果
1. 首先看到 `homeView` 主菜单界面
2. 点击「开始」按钮，主菜单消失，出现 `gameView`
3. 点击暂停按钮，弹出 `pauseView`
4. 游戏结束后，显示 `resultView`

### 9.4 常见问题

| 问题 | 原因 | 解决方法 |
|---|---|---|
| 黑屏，什么都不显示 | Canvas 未挂载 `MainEntry` 组件 | 回到「二、第一步」重新挂载 |
| 点击按钮没反应 | 按钮未绑定事件 | 回到对应章节绑定 Click Events |
| 运行时报错「prefab not found」| 预制体路径不对或不在 resources/ 下 | 检查预制体是否在 `assets/resources/prefab/view/` 下 |
| 方块不显示 | GameNode 节点不存在或命名不对 | 检查 `gameView` 下是否有 `GameNode` 子节点 |
